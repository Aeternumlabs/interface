---
title: Read Functions
description: All view functions — isRegistered, getRecoveryConfig, isRecoveryDue, getTimeUntilRecovery, and more.
---

All functions on this page are `view` — they read state without modifying it and cost no gas when called off-chain. They are safe to call at any frequency from a frontend or indexer.

<FunctionCard
  name="isRegistered"
  signature="isRegistered(address wallet) external view returns (bool)"
  modifier="view"
  description="Returns true if the given address currently has an active, registered vault. Returns false for unregistered addresses, abandoned vaults, and cancelled vaults."
  params={[
    { name: 'wallet', type: 'address', description: 'The address to check.' },
  ]}
  returns="bool — true if the wallet has an active vault, false otherwise."
  example={`const isActive = await vault.read.isRegistered([walletAddress])`}

<FunctionCard
  name="getRecoveryConfig"
  signature="getRecoveryConfig(address wallet) external view returns (RecoveryConfig memory)"
  modifier="view"
  description="Returns the full vault configuration for the given address. This is the primary data-fetch function for frontends — one call returns all vault state. Returns a zero-value struct (all fields at their default values) for unregistered addresses."
  params={[
    { name: 'wallet', type: 'address', description: 'The address to query.' },
  ]}
  returns="RecoveryConfig struct: { backupAddress, inactivityPeriod, lastActivity, balance, isActive, failedRecoveryAttempts, isAbandoned }"
  example={`const config = await vault.read.getRecoveryConfig([walletAddress])
// config.balance          → vault ETH balance (wei)
// config.lastActivity     → Unix timestamp of last interaction
// config.inactivityPeriod → seconds until recovery is eligible
// config.backupAddress    → recipient on recovery
// config.isActive         → vault is live and monitored
// config.isAbandoned      → true after 3 failed recovery attempts`}
/>

<FunctionCard
  name="isRecoveryDue"
  signature="isRecoveryDue(address wallet) external view returns (bool)"
  modifier="view"
  description="Returns true if a vault is currently eligible for recovery — meaning it is active, holds a non-zero ETH balance, and the inactivity period has fully elapsed since the last interaction. Chainlink uses this logic (inlined in checkUpkeep) to identify due wallets."
  params={[
    { name: 'wallet', type: 'address', description: 'The address to check.' },
  ]}
  returns="bool — true if the vault is due for recovery, false otherwise."
  example={`const isDue = await vault.read.isRecoveryDue([walletAddress])
// true  → Chainlink will trigger recovery on the next performUpkeep call
// false → vault is not eligible (timer running, zero balance, or inactive)`}
/>

  modifier="view"

<FunctionCard
  name="getRegisteredWallets"
  signature="getRegisteredWallets(uint256 start, uint256 end) external view returns (address[] memory)"
  modifier="view"
  description="Returns a paginated slice of the registered wallets array. The end index is clamped to the array length if out of bounds, so passing a large end value safely returns everything from start to the last element. Used by off-chain tools and indexers to enumerate active vaults."
  params={[
    { name: 'start', type: 'uint256', description: 'Inclusive start index. Returns an empty array if start >= total registered.' },
    { name: 'end', type: 'uint256', description: 'Exclusive end index. Clamped to array length if out of bounds.' },
  ]}
  returns="address[] — slice of registered wallet addresses from index start to end."
  example={`// Fetch first 100 registered wallets
const wallets = await vault.read.getRegisteredWallets([0n, 100n])

// Fetch all (pass a large end value — it is clamped automatically)
const all = await vault.read.getRegisteredWallets([0n, 1_000_000n])`}
/>

<FunctionCard
  name="getCheckCursor"
  signature="getCheckCursor() external view returns (uint256)"
  modifier="view"
  description="Returns the current position of the rolling cursor — the registry index at which the next checkUpkeep scan window begins. Used by monitoring tools and the docs dashboard to show where in the registry Chainlink is currently scanning."
  params={[]}
  returns="uint256 — the current cursor index into the registered wallets array."
  example={`const cursor = await vault.read.getCheckCursor()

const progress = Number(cursor) / Number(total) // scan progress as fraction`}
/>
